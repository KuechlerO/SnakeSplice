Output Documentation

A simple overview of your given sample input is given. Each row represents a single sample.

Column descriptions:
  1. sample name: The name of the analyzed sample.
  2. sample_directory: Directory, where the data (i.e. FASTA files) for the given samples are stored.
  3. read1: File name of FASTA file for first reads.
  4. read2: File name of FASTA file for second reads.
  5. control: This columns whether the given sample can be used as control sample.
  6. condition: The underlying condition for the given sample.
  7. protocol: The used protocol for the RNAseq library preparation.
  8. stranded: State whether the given sample data is unstranded ("no"), forward stranded ("yes"), or reverse stranded ("reverse").
  9. adaptors_file: Path to FASTA file, where adaptor sequences are stored, which will be used for the cutoff with trimmomatic.
  10. additional_comment: More space for additional comments.
Method:

PEP, or Portable Encapsulated Projects, is a community effort to make sample metadata reusable. PEPs decouple metadata from analysis by providing:

  • a standardized metadata structure;
  • a metadata validation framework;
  • programatic metadata modifiers.

This workflow takes advantage of the PEP metadata structure to provide a standardized way to describe your samples.
